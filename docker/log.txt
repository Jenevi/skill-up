08.08.25
- docker inspect [name or ID]
- docker run -it - запустить контейнер и вывести вывод в терминал: 
  -i - связать стандартный ввод хоста со стандартным вводом контейнера
  -t - связать стандартный вывод хоста со стандартным вводом контейнера
- echo 'qwe' | docker run -i [name or ID] - передать на ввод контейнеру строку qwe
- docker run -p 80:5000 [name or ID] - сопоставить порт 80 хоста порту 5000 в контейнере
- docker run -v /opt/datadir/:/var/lib/mysql [name or ID] - смонтировать каталог /opt/datadir/ хоста в 
  каталог /var/lib/mysql в контейнере

07.08.25
- docker run [name] - команда запуска образа
- docker run -d [name] - запуск в detach mode, когда вывод приложения не связан с консолью.
- docker run --name [name] - запуск контейнера и присвоение ему имени
- docker attach [ID] - прикрепить консоль к конкретному контейнеру.
- docker ps - список запущенных контейнеров и основная информация о них
- docker ps -a - показывает список всех контейнеров.
- docker stop [name or ID] - останавливает контейнер.
- docker rm [name or ID] - удаляет остановленный или завершенные контейнеры.
- docker images - показывает список сохраненных образов.
- docker rmi [name or ID] - удаляет образ.
- docker pull [name] - закачивает образ.
- как только основной процесс в контейнере завершается, контейнер останавливается. К примеру, если база данных   скрашится, контейнер сразу же остановится.
- docker exec {name or ID} [command] - выполняет команду на работающем контейнере.
- docker $(docker ps -aq) - удаляет все остановленные контейнеры.
- docker run -it [name] - запускает контейнер и подключает к нему интерактивную консоль
- docker inspect [name or ID] - подробная информация о контейнере
